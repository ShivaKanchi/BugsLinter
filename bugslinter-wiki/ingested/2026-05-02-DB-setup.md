# 1. Database Choice (Opinionated)

### Recommended Stack

| Concern          | Choice                              |
| ---------------- | ----------------------------------- |
| Primary DB       | **PostgreSQL**                      |
| Search           | **OpenSearch** or **Elasticsearch** |
| Cache / Realtime | **Redis**                           |
| AI/Embeddings    | **Postgres + pgvector**             |

### Why this stack

- PostgreSQL gives:
  - Strong relational guarantees (critical for workflows)
  - JSONB for flexible fields (Jira-style custom fields)
  - `pgvector` for AI similarity search (no need for Pinecone early)

- Avoid MongoDB:
  - Weak joins = painful reporting + workflow transitions

- Avoid over-engineering:
  - Don't introduce CQRS/event sourcing _yet_

# 2. Multi-Tenancy Strategy

### Go with:

**Shared DB, shared schema, tenant isolation via `tenant_id`**

```sql
tenant_id UUID NOT NULL
```

Add this to **EVERY table**.

### Enforce isolation at DB level:

Use PostgreSQL Row-Level Security (RLS)

# 3. Core Data Model (Refined)

You missed some critical abstractions. Here's a corrected and scalable version:

---

## 3.1 Organization

```sql
organizations (
  id UUID PK,
  name TEXT NOT NULL,
  slug TEXT UNIQUE,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
)
```

---

## 3.2 User

```sql
users (
  id UUID PK,
  tenant_id UUID,
  email TEXT UNIQUE NOT NULL,
  name TEXT,
  avatar_url TEXT,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
)
```

---

## 3.3 Membership (CRITICAL - you missed this)

Users belong to organizations via roles.

```sql
memberships (
  id UUID PK,
  tenant_id UUID,
  user_id UUID,
  organization_id UUID,
  role TEXT, -- admin, member, viewer
  created_at TIMESTAMP
)
```

---

## 3.4 Project

```sql
projects (
  id UUID PK,
  tenant_id UUID,
  organization_id UUID,
  name TEXT NOT NULL,
  key TEXT, -- e.g., "ENG"
  description TEXT,
  created_by UUID,
  created_at TIMESTAMP,
  updated_at TIMESTAMP
)
```

---

## 3.5 Issue

This is your core domain object.

```sql
issues (
  id UUID PK,
  tenant_id UUID,
  project_id UUID,
  title TEXT NOT NULL,
  description TEXT,
  status_id UUID,

  reporter_id UUID,

  created_at TIMESTAMP,
  updated_at TIMESTAMP
```

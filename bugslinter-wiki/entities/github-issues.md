# GitHub Issues

## Summary

GitHub Issues is a lightweight, free issue tracking system integrated with GitHub code hosting. It emphasizes code-centric workflows but lacks advanced project management features compared to dedicated platforms.

## Key Facts

- **Core Features**: Issues, labels, milestones, project boards (basic Kanban), free for developers; tightly integrated with Git
- **Advantages**: Free, native Git integration, tightly integrated with code
- **Limitations**: Lacks structured workflows (no native sprints), limited project management features (no reporting/dashboards), no built-in complex permission schemes
- **AI Features**: GitHub Copilot Chat for issues (beta); experimental AI support via Marketplace apps
- **Target Users**: Open-source developers, individual developers, small teams hosted on GitHub
- **Ecosystem**: GitHub Marketplace for extensions and integrations

## Implications

- **Low Friction**: Free tier removes adoption barrier; many small teams start with GitHub Issues
- **Lock-In**: Deep Git integration means switching costs are high; competing tools must offer compelling reasons to move
- **Enterprise Gap**: GitHub Issues lacks features for large teams; enterprises need Jira-like depth
- **Competitive Opportunity**: New tools can offer GitHub Issues' simplicity + missing advanced features (reports, sprints, complex workflows)

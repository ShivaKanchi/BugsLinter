# Linear

## Summary

Linear is a developer-first issue tracking and project management tool emphasizing lightning-fast UI, tight Git integration, and keyboard-driven workflows. It appeals to engineering-focused teams seeking speed and simplicity.

## Key Facts

- **Core Features**: Cycles (sprints), roadmaps, fast keyboard shortcuts, superb Git/GitHub integration with auto-linking, keyboard-first UI
- **Strengths**: Lightning-fast performance, excellent UX for developers, smooth GitHub integration, auto-linking PRs and commits
- **Target Users**: Fast-moving engineering teams, startups, developer-first organizations
- **Customization**: Low (more opinionated), limited custom fields compared to Jira
- **AI/ML**: Minimal AI support; basic custom fields for ML projects
- **Pricing**: ~$10/user/month
- **Weaknesses**: Less suitable for non-technical stakeholders, limited customization, smaller ecosystem

## Implications

- **Speed Advantage**: Linear's performance is a key differentiator; users appreciate snappy UX
- **Developer Satisfaction**: Linear's focus on keyboards, Git integration, and simplicity appeals to engineers
- **Competitive Threat**: Linear demonstrates that simpler, faster tools can capture market share from feature-rich platforms like Jira
- **Market Niche**: Developer-first positioning is clear; competitors must choose between Jira-like customization or Linear-like speed
